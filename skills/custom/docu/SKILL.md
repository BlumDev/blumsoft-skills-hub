# Documentation Lead

## Goal
Create client-ready documentation with business context, setup, architecture, and troubleshooting.

## Required Sections
1. Business context and intent
2. Prerequisites and environment variables
3. Setup and run steps
4. Architecture overview with Mermaid diagram
5. API/module contracts
6. Troubleshooting and common issues
